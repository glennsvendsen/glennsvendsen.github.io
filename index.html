Problem
Misconfigurations were frequent and hard to detect. Operators often didn't know something was wrong until it had real-world consequences — incorrect pricing live in market, support escalations, and eroded trust in the system.
Approach
Ran a deep-dive root cause analysis across misconfiguration reports to find patterns. Most errors fell into predictable categories: price spikes above the median or adjacent-unit threshold, zero prices on active periods, cap fields set to zero, date ranges where validFrom exceeded validTo, and pricing tables that had expired or left gaps in coverage. Designed a background validation tool that checks 33 parameters silently on every save — surfacing issues as non-intrusive notifications, never blocking the workflow.
Outcome
Misconfigurations reduced by 36% year-on-year. Operators get immediate visibility into potential issues without friction. Errors are caught at the source before reaching live pricing — and acknowledged issues leave a clear audit trail.

Problem
Configuration experts managing parking across major US cities faced a scale problem: a single nationwide holiday across thousands of zones meant one missed configuration could result in drivers being charged when parking should have been free.
Approach
Built a working prototype with AI during the stakeholder call itself — validated live, iterated on the spot, and confirmed as the right solution within the hour. Handed off to engineering the same day to build it securely and properly. The tool lets users select multiple holiday dates, configure time windows and durations, then fire price calls across the full zone estate. A "Check Free Holiday" mode highlights any zone returning a price above zero.
Outcome
Zero holiday-related pricing incidents reported since release. The tool's flexibility made it useful beyond holidays — it's now a general-purpose pricing QA tool used by configuration experts across markets.

Problem

Approach
Led systematic gap discovery across the full platform scope — calculation engine, configuration UI, tariff types, and operator workflows. Identified 35+ distinct gaps. Problem
Operators had no ability to update their own pricing. Every rate change — however routine — required a configuration expert. The question was whether a self-service product was viable, and if so, what exactly would it need to cover to be useful to the breadth of the operator base.
Approach
Pulled operator and tariff data from Snowflake for 5,650+ EU operators. Built a 0–25 complexity scoring model across 39 configuration dimensions, segmenting the operator base into four tiers. Mapped all 40 tariff fields for self-serviceability. Modelled a feature milestone roadmap showing the cumulative operator coverage unlocked at each tier — revealing that 8 feature tiers alone would cover 71% of all operators.
Outcome
Delivered a data-backed MVP scope with quantified coverage at every feature milestone. Identified that 52% of operators are immediately viable candidates. Price changes — the most requested operator action — emerged as the clearest, highest-value starting point for any self-service feature.
